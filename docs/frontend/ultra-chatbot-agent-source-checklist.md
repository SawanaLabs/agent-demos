# Ultra Chatbot Agent Source Checklist

## Source Snapshot

- Repository: `https://github.com/vercel/ai-chatbot`
- Snapshot commit: `2becdb4a56e7683ae08aef927cec1c6c52dfad5e`
- File count: 180 tracked files
- Preferred source access: `pnpm dlx opensrc path vercel/ai-chatbot#2becdb4a56e7683ae08aef927cec1c6c52dfad5e`
- Port acceptance is frozen to this snapshot. If upstream changes, create a separate upstream refresh task and regenerate or diff this checklist.

## Checklist Rule

- Keep this checklist aligned to the source snapshot used for the port.
- Convert source files into implementation checklist items before or during the relevant vertical TDD slice.
- Mark a file checked only when every needed function, component, route behavior, schema contract, or test fixture from that file has been implemented, adapted, or explicitly deferred with a reason.
- If a source file is intentionally outside this repository's boundary, keep the item unchecked until the defer reason is written next to it.
- Final acceptance for the Ultra Chatbot Agent requires every source-tree checklist item to be checked.
- Final user signoff for this demo uses this checklist as the gate. Partial slices, green tests, or a working UI do not count as done while unchecked source items remain.

## Checked Meaning

- Checked means the source file has been audited and fully disposed for this repository.
- If the file should be ported, check it only after the needed behavior is implemented or adapted.
- If this repository already has equivalent behavior, check it only after naming the equivalent local file, module, or convention.
- If the file is unnecessary for this demo, check it only after writing the skip reason.
- If the file conflicts with Visitor Owner ownership, demo boundaries, infrastructure, or release safety, check it only after writing the defer reason.
- Do not use a bare checkmark for skipped, deferred, or already-covered source files.
